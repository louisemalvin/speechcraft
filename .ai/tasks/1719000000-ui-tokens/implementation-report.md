# Implementation Report: UI Unit A — Design Tokens & Metadata

## Outcome

All three deliverables completed successfully. The build (`pnpm --filter frontend build`) succeeds with no CSS/Tailwind errors.

## Files Changed

- **Expanded `@theme inline` block** with:
  - 28 `--color-*` tokens covering surface, text, accent, status, theme, and overlay categories (hex values match spec table exactly).
  - 6 `--radius-*` tokens (`sm`/`md`/`lg`/`xl`/`2xl`/`full`).
  - 7 `--animate-*` tokens referencing the `@keyframes` below.

### 2. `packages/frontend/src/app/layout.tsx`

- **Preserved unchanged**: `manifest: "/manifest.json"`, `appleWebApp` config, `themeColor`, `viewport` export, `RootLayout` component, service worker script.

### 3. `packages/frontend/public/favicon.svg` (new)
- Microphone icon on a dark circular background (slate-950 background, indigo-500 glyph).
- Valid SVG: `viewBox="0 0 32 32"`, well-formed XML, self-contained.

## Decisions

- **Token placement**: All `@theme inline` tokens go inside the existing `@theme inline` block (after existing entries). Keyframes go below the block as standard CSS — this is required by Tailwind v4 (keyframes must be in standard CSS, not inside `@theme`).
- **Favicon format**: SVG chosen over ICO for simplicity and modern browser support. Not referenced in `metadata.icons` — Next.js automatically serves `favicon.svg` from `public/` when present.
- **No inline style removal**: The `<style>` blocks in `page.tsx` and `speaker/page.tsx` are NOT removed (explicit non-goal per task spec — will be handled by page-refactor units).
