## Residual Risks

- **CSS cascade with inline `<style>` blocks**: The `@keyframes` definitions now exist in both `globals.css` and the inline `<style>` blocks in `page.tsx`/`speaker/page.tsx`. The Tailwind v4-generated utility classes (e.g., `animate-fade-in-up`) will use the global keyframes. The duplicate inline definitions are harmless per CSS cascade (same values, last-defined wins). They will be removed by future page-refactor units (explicit non-goal per spec, Open Question #2).

- **`themeColor` migration**: Next.js 16 recommends moving `themeColor` from `metadata` to `viewport`. This is a pre-existing issue tracked separately, not in scope for this unit.
